## Feature Notes (Latest)

- **Per-nivo lesson books (PDF):** Books are managed as a **static nivo-level set** (`Ilmihal 1.pdf` ... `Ilmihal 5.pdf`) under `server/prisma/seed-assets/nivo-books`, not as lesson attachments. Lessons keep only lesson data (`title`, `nivo`) and do not upload/replace files. The app reads available files by nivo and shows them in the Lessons nivo header and child details.
- **Nivo book preview contract:** All authenticated roles can preview a nivo PDF through `GET /nivo-books/:nivo/preview`. The list endpoint `GET /nivo-books` returns only files that exist in the static folder, with metadata (`nivo`, `originalName`, `mimeType`, `sizeBytes`, `updatedAt`).
- Shared **pagination** controls (`PaginationControls`) show **chevron** icons beside localized **Previous** / **Next** and a translated **Page x of y** label (users, children, activities, homework queue). List panels pass **local `page` state** into the pager (not the API’s `page` field from cached placeholder data) so **Next** / **Previous** stay correct while queries with `placeholderData` refetch. **`DebouncedSearchInput`** keeps the debounce timer on **`localValue` only** (callback via ref) so parent re-renders do not re-fire the debounce; list toolbars reset **page to 1** only when the debounced search string **actually changes**.
- **User settings (`/app/settings`):** Signed-in users open **Settings** from the **user menu** (avatar at the bottom of the sidebar); the page stores **interface language** via `PATCH /auth/me` (`preferredLanguage`: `en` | `sv` | `bs`) and includes a **More settings** placeholder for future preferences. `GET /auth/me` returns a safe profile shape (no password hash). After sign-in, the app syncs UI language from the account when the stored session omits `preferredLanguage` (older local storage). Sidebar language changes persist the same way.
- **Account invitation email:** Greeting always opens with **Esselamu alejkum** (with the invitee’s first name when present). Pending users receive a branded verification message with a header logo, a primary **Activate account** CTA, a copy-paste fallback link, and a plain-text part for mail clients that strip HTML. **Logo URL resolution:** `EMAIL_LOGO_URL` (full image URL) if set; else when `FRONTEND_URL` is a local dev origin (`localhost`, `127.0.0.1`, etc.), `{EMAIL_PUBLIC_WEB_URL}/branding/logo-small.svg` so recipients can load the asset; otherwise `{FRONTEND_URL}/branding/logo-small.svg`. Subject and body are localized to **English**, **Swedish**, or **Bosnian** from the invitee’s stored **`preferredLanguage`** (`UserPreferredLanguage` on `User`). Admins set **Account language** in the create/edit user dialog (defaults to the admin’s current UI language on create). **API:** `POST /users` and `PATCH /users/:id` accept `preferredLanguage` (`en` | `sv` | `bs`); `POST /auth/invite-token` accepts optional `preferredLanguage` (defaults to `en`). For widest email-client compatibility (remote images and SVG support vary), point `EMAIL_LOGO_URL` at a hosted PNG if needed.
- **Diplomas (PDF):** On **Community → Diplomas**, `ADMIN` and `SUPER_ADMIN` configure **per-community** diploma settings on two **cards**: **PDF template** (replace/upload background PDF, save to community, remove custom; **active template** line shows default filename or saved/pending source because the browser file input cannot display the built-in default; **amber alert** if the default asset or stored PDF fails to load, with distinct copy for missing default file vs missing custom template) and **Text positions and style** (default imam line—when unset, suggested from the first assigned community admin as `First ef. Last`—click-to-place layout, advanced typography including **child name typeface**—sans vs script for the **full name line only**; nivo, date, and imam stay Noto Sans—**sample output** with debounced rebuild and **cached** diploma font load). Layout uses **X/Y** in PDF points (optional X per line, **Y** baseline from bottom, or centered when X is empty). The sample canvas uses the same `pdf-lib` path as real batches. **Generate diplomas** opens a **dialog** for batch fields: **nivo** filter, **optional custom nivo line** (shown when a specific nivo is selected; replaces the default “Nivo N” text on every page in that run), **ceremony date**, imam override, child multi-select, and **Generate & download** of a **single merged PDF** (one page per child), with the usual **dialog header + scrollable body + footer** (Cancel and primary action together in the footer). `SUPER_ADMIN` sees only children belonging to the **current** community in this tab (API may return wider lists). Code defaults live in [`diploma-layout.ts`](web/src/features/diplomas/diploma-layout.ts) and merge with saved JSON when present. Generation runs **in the browser** (`pdf-lib`); **generated PDFs are not stored**. **API:** `PATCH /communities/:id/diploma-settings`, `PUT/GET/DELETE /communities/:id/diploma-template` (PDF bytes never included in normal community JSON). Default asset: `web/public/diplomas/default.pdf`. List capped at **100** children per request. Fonts: `web/public/fonts/NotoSans-Regular.ttf`, optional `web/public/fonts/DancingScript-wght.ttf` for script name lines, + `@pdf-lib/fontkit`. **Template preview** uses `pdfjs-dist`; the Vite build emits the PDF worker as **`assets/pdf.worker-*.js`** (not `.mjs`) so static hosts serve it as JavaScript—avoiding `application/octet-stream` MIME failures on nginx/Render. **`Dockerfile.web`** also adds `mjs` to nginx `mime.types` when missing, as a fallback.
